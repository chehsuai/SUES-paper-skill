# 学术论文自动生成 Skill

## 概述

本 skill 从 Word 模板出发，自动完成学术论文的撰写与排版。核心能力：
- 解析模板 → 提取选题、格式、字数要求
- 搜索真实文献 → 确保每条引用在知网/Google Scholar 上可查
- 撰写论文 → 5000-8000 字，结构完整，观点明确
- 生成 Word → 上标引用 + 书签交叉引用，Ctrl+点击可跳转

---

## Phase 1: 读取并解析模板

### 1.1 安装依赖

首先确保 `python-docx` 已安装：

```bash
pip install python-docx -q
```

### 1.2 提取模板内容

用 Python 读取模板文件，提取所有段落文本和样式：

```python
from docx import Document
doc = Document('模板文件路径.docx')
with open('template_text.txt', 'w', encoding='utf-8') as f:
    for i, para in enumerate(doc.paragraphs):
        if para.text.strip():
            f.write(f'[{i}] Style: {para.style.name} | {para.text}\n')
```

### 1.3 识别关键信息

从模板中识别以下要素：
- **论文主题/选题方向** — 通常在"具体要求"段落中
- **字数要求** — 如 "5000-8000字"
- **格式要求** — 字体、字号、行距、标题层级、缩进等
- **参考文献格式** — 如 GB/T 7714
- **课程信息** — 课程名称、学院、指导教师（封面用）
- **特殊要求** — 引用率限制、案例要求等

### 1.4 与用户确认

如果模板中有以下不确定信息，**必须向用户确认**：
- 学号和姓名（模板中通常是占位符 `xxx` 或 `M07xxxxxxx`）
- 具体选题（如果模板允许多个方向）
- 用户的专业方向（用于选题建议）

**重要：不要猜测用户的个人信息。** 模板中的 `xxx` 占位符保留即可，提醒用户自行替换。

## Phase 2: 确定选题与结构

### 2.1 选题原则

- 在模板给出的主题范围内，选择一个**具体、可深入**的切题角度
- 必须能结合**具体案例**展开论述（模板通常明确要求这一点）
- 选题要能覆盖用户专业方向的关键维度（如设计学 → 视觉/空间/交互）
- 副标题用"以XX为例"的形式锁定研究对象

### 2.2 论文结构模板

```
封面页（课程名称、学院、学号、姓名、指导教师）
内页标题 + 学号姓名
摘要（200字以内）+ 关键词（3-5个，分号分隔）
正文：
  一、绪论
    （一）研究背景与意义
    （二）研究对象与方法
  二、理论基础（2-3个小节）
  三、案例分析/策略分析（3-4个维度）
  四、经验与启示/建议
  五、结论
参考文献（5-9条）
```

## Phase 3: 搜索真实参考文献

**这是最关键的一步。所有参考文献必须真实可查。**

### 3.1 搜索策略

使用 `WebSearch` 工具进行多轮搜索：

1. **中文文献（知网）**：搜索 `"选题关键词" 知网 学术论文 期刊`
2. **补充搜索**：改变关键词组合，覆盖理论、案例、方法等不同维度
3. **核实**：确认搜索结果的期刊名称、年份、期号、页码完整

### 3.2 文献选取标准

- 优先选择 **核心期刊**（CSSCI/北大核心）论文
- 至少包含 2 篇与案例对象直接相关的文献
- 包含理论框架类和实证研究类两种类型
- 英文经典著作（如 Pine & Gilmore 的《体验经济》）可以保留，它们真实可查
- **绝对禁止虚构文献**：作者名、期刊名、年份、卷期号必须与搜索结果一致

### 3.3 文献数量

- 5000-8000 字论文：5-9 条参考文献
- 正文中确保每条文献至少被引用 1 次

## Phase 4: 撰写论文内容

### 4.1 写作要求

- **语言**：学术化中文，避免口语，避免空洞套话
- **字数**：严格控制在要求的范围内（通过 Python 脚本统计中文字符数）
- **原创性**：观点明确，有自己的见解，不可泛泛而谈
- **逻辑**：层层推进，每个论点有论据支撑
- **引用**：关键论断后用 `[n]` 标注对应文献

### 4.2 各部分写作要点

**摘要（200字以内）**：研究背景 1 句 + 研究对象/方法 1 句 + 主要发现 1-2 句 + 结论/意义 1 句

**绪论**：
- 背景从宏观（时代趋势）→ 中观（行业变化）→ 微观（案例引入）
- 明确指出现有研究的不足（你的研究填补什么空白）
- 方法部分简述研究路径

**理论基础**：不要简单罗列定义，要有自己的梳理和批判性思考。将不同学者的观点进行对话和整合。

**案例分析**：这是论文核心（占 40-50% 篇幅）。至少分 3-4 个维度深入分析，每个维度先阐述策略是什么，再分析它如何实现品牌传播目的，最后给出评述。

**启示/建议**：基于案例分析提炼规律，提出有指导意义的原则或建议，不可空泛。

**结论**：概括核心发现 + 研究局限 + 未来方向

### 4.3 引用规范

- 引用标记统一用方括号上标格式，如 `[1]`、`[3][4]`
- 引用放在被引内容的句末标点之前
- 每个自然段不宜引用过多（1-3 处为宜）

## Phase 5: 生成 Word 文档

### 5.1 使用配套脚本

本 skill 配套的 `generate_paper.py` 脚本负责 Word 文档的生成。它会自动处理：

- 封面页和内页的格式排版
- 正文宋体小四、1.5倍行距、首行缩进2字符
- 一级标题（一、二、三）、二级标题（（一）（二）（三））、三级标题（1. 2. 3.）顶格，不加粗
- 引用标记上标显示（`<w:vertAlign superscript>`）
- 引用标记与参考文献条目之间的书签+超链接交叉引用（`<w:bookmarkStart>` + `<w:hyperlink>`）
- 参考文献宋体五号

### 5.2 脚本核心函数

```python
# 正文段落 — 自动将 [n] 转为上标超链接
add_body_para(text_with_refs)

# 参考文献条目 — 带书签锚点
add_ref_entry(ref_text, ref_num)

# 标题 — 顶格，宋体小四，不加粗
add_heading_para(text, level)
```

### 5.3 交叉引用实现原理

1. 参考文献每条追加 `<w:bookmarkStart w:name="refN">` 和 `<w:bookmarkEnd>`
2. 正文中每个 `[N]` 包裹在 `<w:hyperlink w:anchor="refN">` 中
3. Word 中 Ctrl+点击引用号 → 跳转到对应文献条目

### 5.4 执行

将论文内容写入 `generate_paper.py` > 运行脚本 > 生成 `.docx` 文件。

**注意**：如果目标文件名已被 Word 打开锁定，改用临时文件名（如 `paper_output.docx`）。

## Phase 6: 验证与清理

### 6.1 质量检查

生成后确认：
- [ ] 中文字符数在要求范围内
- [ ] 每篇参考文献在正文中至少被引用一次
- [ ] 交叉引用书签数量 = 参考文献数量
- [ ] 参考文献格式正确（期刊 [J]、专著 [M]、学位论文 [D] 等标识）
- [ ] 模板中的括号注释文字已全部删除
- [ ] 无虚构文献，全部可查

### 6.2 清理

删除临时文件（`template_text.txt`、`generate_paper.py` 等），只保留最终的 `.docx` 文件。

## 常见问题

**Q: 模板是中文文件名，Python 读取乱码？**
A: 将提取的文本写入 UTF-8 编码的 txt 文件再读取，避免终端编码问题。

**Q: 生成的 docx 文件被 Word 占用无法覆盖？**
A: 换个文件名保存，或提醒用户关闭 Word 后重新生成。

**Q: 用户要求在论文中使用特定理论框架？**
A: 如果用户指定了理论框架（如 S-O-R 模型、体验营销 5 维度等），优先使用并在参考文献中补充相关文献。

**Q: 引用率如何控制？**
A: 用自己的语言重新组织已有知识的表述，避免直接复制文献原文。每个观点的表述应融入自己的理解和分析框架。
